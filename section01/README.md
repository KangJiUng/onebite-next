## 섹션1: Next.js를 소개합니다

- Next.js = React.js 전용의 웹 개발 “Framework”
- React.js = UI 개발을 위한 JavaScript “Library”

- Framework vs Library

  - 기능 구현의 주도권이 누구에게 있는가?
    - Framework: 개발자에게 없음 ⇒ 프레임워크가 제공하는 기능을 이용하거나 허용하는 범위 내에서만 추가적인 도구를 사용
    - Library: 개발자에게 있음 ⇒ 기능 구현을 원하는 방향으로 진행

- 사전렌더링

  - 브라우저의 요청에 대해 서버측에서 사전에 렌더링이 완료된 HTML을 응답해주는 렌더링 방식
  - 기존의 리액트 앱이 따르는 CSR 방식의 단점을 효율적으로 해결하는 기술
    - 요청이 시작된 시점으로부터 콘텐츠가 실제로 화면에 나타나기까지 걸리는 시간(First Contentful Paint, FCP)는 웹페이지의 성능을 대표할 정도로 굉장히 중요한 지표 → CSR은 이 FCP가 느림
    - 반면에, Next.js가 택하는 SSR 방식은 서버측에서 미리 렌더링이 완료된 HTML을 주기 때문에 FCP가 짧음

- 수화(Hydration)
  - 기존의 HTML은 아무런 상호작용도 할 수 없으나, 즉시 서버는 이 웹페이지에 자바스크립트 코드 즉, 리액트 앱을 번들링해서 보내주게되는 과정
  - 메말라 있던 HTML 요소들에 자바스크립트라는 물을 뿌려주는 것과 비슷하다하여 생긴 말

→ 이후부터는 CSR과 동일하게 브라우저가 직접 자바스크립트 코드를 실행하여 컴포넌트만 교체
